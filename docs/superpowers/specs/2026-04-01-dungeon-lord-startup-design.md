# 魔王創業 - 遊戲設計文件

## 概要

Game Jam 作品。玩家扮演魔王經營地城，建設房間、佈置陷阱、招募怪物，吸引勇者來犯後擊敗並俘虜他們，透過刑求轉化為邪惡勢力壯大自己的軍團。

## 專案限制

| 項目 | 規格 |
|------|------|
| 開發時限 | 兩週 |
| 團隊 | 一人（Claude Code 輔助） |
| 引擎 | Phaser |
| 平台 | 手機網頁（直式 Portrait） |
| 美術風格 | 卡牌 UI + 像素風 sprite（nano banana 生成） |
| MVP 規模 | 1 關、怪物 5 種、英雄 5 種、房間 5 種、陷阱 5 種 |

## 核心結構：混合制

- **單局制**：每局地城重置，翻完所有牌或魔王死亡即結束
- **永久保留（Meta-progression）**：魔王等級、已解鎖的房間/怪物種類、圖鑑

---

## 核心遊戲循環

### 主畫面：翻牌矩陣

每局開始，玩家面對一個覆蓋牌組成的矩陣（例如 3x5 = 15 張）。每回合翻一張牌，隨機揭示事件：

| 牌面類型 | 效果 |
|----------|------|
| 普通戰鬥 | 一般英雄來犯，進入防禦階段 |
| 精英戰鬥 | 強力英雄來犯 |
| 地宮 | Boss 級英雄 |
| 活動 | 隨機事件：獲得怪物、建設卡、陷阱卡、資源 |
| 寶箱 | 直接獲得金幣或道具 |
| 商店 | 花金幣購買指定卡牌 |

翻到戰鬥類 → 切到地城地圖進行防禦。翻到非戰鬥類 → 結算獎勵後繼續翻牌。玩家可在翻牌之間隨時進入地城地圖佈置。全部翻完或魔王死亡 = 本局結束。

## 地城地圖

### 畫面呈現

- **直式可捲動大地圖**：地圖比螢幕大，手指拖拽平移瀏覽
- **羊皮紙底圖**質感，地城是畫在上面的藍圖
- **頂部 HUD + 底部操作列固定**不隨地圖捲動

### 地圖結構（由上到下）

```
頂部：英雄傳送門（1-2 個，依關卡而定）
  │
  ├─ 格子串聯路徑（每局隨機生成拓撲）
  │   各格子間有鎖鏈/線段連結
  │   英雄沿此路徑由上往下推進
  │
底部：地城之心（魔王所在，可強化）
```

- 每局地圖的格子串聯拓撲不同
- 傳送門到地城之心一定有至少一條完整路徑
- 格子內有符文式圖標標示房間/陷阱類型
- 怪物以像素角色顯示在格子內部

## 佈局系統

### 抽卡機制

- 開局免費三選一 x1
- 之後每次三選一需花費金幣，費用遞增：
  - 免費 → 50 → 100 → 150 → 250 → 350 → 500
- 三選一卡池包含房間卡和陷阱卡（混合池）
- 抽到的卡進入手牌區（畫面上某個區域顯示累積的牌組）

### 格子結構（每格最多 3 層）

```
├─ 房間卡（底層）：決定格子類型，疊加同類型卡 = 升級房間
├─ 陷阱卡（中層）：附加陷阱效果，疊加同類型卡 = 升級陷阱
└─ 怪物（頂層）：1 隻我方單位，獨立 UI 操作放置
```

### 放置流程

1. 抽卡後卡牌進入手牌區
2. 點擊格子 → 從手牌選卡放上去
3. 放同類型卡到已有卡的格子 = 強化升級
4. 每格只能有一種房間 + 一種陷阱
5. 怪物透過獨立 UI 從怪物清單拖放到格子上，每格限 1 隻

### 怪物-房間配對強化

怪物有偏好的房間類型，放在對應房間上可獲得：
- 攻擊/防禦數值加成
- 技能升級為強化版招式

---

## 防禦階段

### 觸發

翻牌翻到戰鬥類（普通/精英/地宮）時觸發。

### 英雄推進

- 英雄從傳送門分梯次進入（不是全部同時）
- 沿預設路徑由上往下逐格推進
- 每格觸發順序：
  1. 陷阱效果先觸發（即時傷害/減速/控制）
  2. 格子上的怪物與英雄自動戰鬥（雙方依 CD 輪流攻擊）
- 英雄 HP 歸零 → 擊敗
- 怪物 HP 歸零 → 該格失守，英雄繼續前進
- 英雄走到底部 → 與魔王/地城之心戰鬥

### 魔王技能

- 防禦階段唯一的玩家主動操作
- 手動點擊施放，有 CD
- 對指定格子/區域造成效果

### 結束條件

- 所有英雄被擊敗 → 勝利，結算收穫
- 魔王 HP 歸零 → 本局結束，結算永久進度

---

## 收穫與俘虜系統

### 戰鬥結算

- 依擊敗英雄數量和強度結算金幣
- 被擊敗的英雄有 **20% 機率**被俘虜
- 俘虜進入暫存池

### 多餘俘虜處理

- 可直接「榨取」換資源（金幣/其他）
- 玩家自行決定誰送去刑求室轉化、誰直接榨取

---

## 刑求室（Pikmin Bloom 花盆式設計）

### 結構

- 獨立 UI 畫面，類似 Pikmin Bloom 的花盆介面
- 初始 2 個欄位（刑具台）
- 花費資源解鎖更多欄位，費用遞增

### 轉化流程

1. 從俘虜暫存池拖一個英雄到刑具台欄位
2. 每個英雄有不同的轉化成本（依強度）
3. 轉化進度：每擊殺一個英雄 → 推進**所有**欄位的進度
4. 進度條滿 → 轉化完成 → 獲得該英雄的邪惡版怪物（保留部分原始技能）
5. 轉化後的怪物加入我方怪物清單，可放置到格子上

### 設計意圖

打越多仗 → 刑求室裡的轉化越快 → 怪物越多 → 打得更好。形成正向循環。

## 華麗度系統

- 華麗度是地城的整體數值，由房間等級和種類累計
- 華麗度越高 → 吸引越強的英雄（更高機率出精英/地宮牌）
- 形成決策拉扯：華麗度高 = 更好的俘虜來源，但也更難防禦

## 視覺風格指南（Visual Style Guide）

### 整體風格：Dark Gothic Fantasy + Pixel Art

融合暗黑哥德奇幻與像素風，營造「邪惡但有趣」的魔王經營氛圍。

| **暗紅-陷阱** | `#8B0000` | 陷阱格子邊框、火焰/傷害 |
| **魔法紫** | `#8a2be2` | 魔王相關、特殊房間、魔法效果 |
| **冰藍** | `#4a9eff` | 英雄陣營、冰屬效果 |
| **毒綠** | `#2ecc71` | 毒屬效果、治療、正面增益 |

| 用途 | 字體 | 備註 |

| **標題/數值** | Press Start 2P | 像素風，用於傷害數字、等級、標題。注意此字體偏寬偏大，需縮小使用 |
| **內文/描述** | VT323 | 像素等寬體，卡牌效果描述、UI 文字 |
| **中文內文** | Noto Sans TC | 卡牌中文名稱、技能描述 |

```css
@import url('https://fonts.googleapis.com/css2?family=Press+Start+2P&family=VT323&family=Noto+Sans+TC:wght@400;700&display=swap');
```

### 像素角色規格

- **風格參考**：Dungeon Maker 的擬人化像素角色（大頭小身、動作姿勢、武器配件明確）
- **尺寸**：角色 sprite 64x64px 或 128x128px，格子內顯示約格子 70% 大小
- **怪物色調**：暗色系（紫/紅/黑/深藍）
- **英雄色調**：亮色系（白/金/藍/綠）
- **轉化後怪物**：保留英雄輪廓但色調翻轉為暗色
- **生成工具**：nano banana（Gemini Imagen），使用以下 prompt 模板：

```
16-bit pixel art character sprite, chibi proportions (large head small body),
[CHARACTER_NAME], [DESCRIPTION], dark fantasy style,
transparent background, action pose, detailed pixel shading,
game asset, RPG monster/hero design
```

### UI 元素風格

**卡牌**
- 材質：羊皮紙漸層底色（`#f5e9d3` → `#e8d5b8`）
- 邊框：深棕 `#8a7a5a`，2px solid，圓角 4px
- 結構（由上到下）：符文圖案區（40%）→ 名稱（粗體）→ 類型+等級 → 效果描述（關鍵字高亮）→ 星級
- 房間卡符文色：紅色調 `#8B0000`
- 陷阱卡符文色：紫色調 `#6a3a8e`
- 選中狀態：金色外框 `#f0c040` 3px + 輕微放大 scale(1.05)

**地圖格子**
- 大小：90x90px
- 已建房間：淡色填充 + 房間類型符文圖標（SVG 筆觸風）
- 已放陷阱：格子右上角小圖標
- 有怪物：像素角色在格子中央
- 空地：虛線邊框 `#8B451366`，灰色問號
- 顏色編碼：紅邊框=陷阱房、紫邊框=特殊房、深棕=一般
- 連結路徑：格子間 SVG 線段，`#8B4513` 3px，帶小圓環鎖鏈裝飾

**HUD（頂部/底部）**
- 背景：`#1a1510` 透明度 90%，帶漸層消失效果
- 文字：`#F8FAFC`
- 資源圖標：HP 紅心、金幣、天數，使用 SVG 圖標（非 emoji）
- 底部操作按鈕：圓角 6px，半透明有色底 + 同色邊框

**刑求室**
- 刑具台底座：圓形，暗灰 `#2a2018`，邊緣鎖鏈/鏽鐵裝飾
- 進度條：綠 `#2ecc71` → 紅 `#e74c3c` 漸變
- 鎖定欄位：灰色 + 鎖頭 SVG
- 俘虜：英雄角色被鎖鏈綁住的變體 sprite

**翻牌矩陣**
- 覆蓋牌面：粉紫色底 + 白色問號 + 邊角閃光粒子
- 翻開動畫：Y 軸 3D 翻轉 300ms ease-out
- 已翻牌面：事件類型像素風插圖 + 底部黑色標籤

### 動畫規範

| 動畫 | 時長 | 緩動 | 說明 |
|------|------|------|------|
| 翻牌 | 300ms | ease-out | Y 軸 3D 翻轉 |
| 卡牌選中 | 150ms | ease | scale(1.05) + 金色外框 |
| 傷害數字彈出 | 600ms | ease-out | 從格子向上浮動 + 淡出 |
| 英雄移動 | 400ms/格 | linear | 沿路徑線段平移 |
| 陷阱觸發 | 200ms | ease-in | 格子邊框閃光 + 粒子效果 |
| 轉化完成 | 800ms | ease | 色調翻轉 + 閃光爆發 |
| 地圖拖拽 | 即時 | - | 跟隨手指，慣性減速 |

### 音效方向（如有時間）

- 翻牌：紙張翻動音
- 陷阱觸發：對應屬性音效（火焰爆裂/冰凍碎裂/毒液滴落/石頭崩塌/箭矢射出）
- 英雄擊敗：悶哼 + 倒地
- 俘虜成功：鎖鏈聲
- 轉化完成：邪惡笑聲 + 魔法音效

## 技術架構（概要）

### Phaser 核心模組

- **MapScene**：可捲動地城地圖，格子系統，怪物放置
- **CardScene**：翻牌矩陣主畫面
- **BattleScene**：防禦階段戰鬥演出
- **TortureScene**：刑求室 UI
- **HUD**：頂部狀態列 + 底部操作列（固定層）

### 資料結構

- 格子：2D Array，每格儲存 `{ room, trap, monster, position, connections[] }`
- 路徑：格子間的有向圖（adjacency list）
- 手牌：卡牌陣列
- 俘虜池：英雄物件陣列
- 刑求室：欄位陣列，每欄位有進度值

### 狀態管理

- 全域 GameState 物件管理跨場景資料
- 永久進度存 localStorage

---

## 擴充預留

所有系統採輕量 MVP 實作，但架構預留擴充彈性：

- **怪物/英雄**：資料驅動（JSON 定義），新增種類只需加資料
- **房間/陷阱**：同上，不硬編碼
- **地圖拓撲**：路徑生成演算法獨立，可調整複雜度
- **翻牌矩陣**：事件類型以 config 定義，可新增
- **刑求室欄位**：數量和費用以 config 控制
- **Meta-progression**：解鎖條件和升級效果以資料表管理
